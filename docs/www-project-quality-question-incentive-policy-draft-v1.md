## 1. 為什麼需要「優質提問激勵」？

大家想知道（WWW Project）若只限制「每天能發幾題」，卻不區分 **清楚、中立、有參與價值** 的問卷與 **洗版、誘導、重複、誤導** 的問卷，長期會出現：

- 首頁與探索被低品質內容佔滿；
- 使用者對公開結果不信任；
- 發起者用聳動標題換取流量，卻在收票規則下仍傷害平台公平（例如收票期間不應靠票數訊號操弄，但 **題目品質** 仍會影響參與與治理成本）。

**激勵制度的目的** 是：

1. **鼓勵** 清楚、可理解、符合分類與資格設定的好問題；
2. **限制** 濫發、刷分、協同灌票、政治／高風險操弄；
3. **與信任分級銜接**（日後），但不以「按讚數」作為唯一品質指標；
4. **符合** 三項 MVP 核心原則與 Raw Option Linkage Ban——激勵計算 **不得** 引入可重建「誰選了哪一選項」的 durable 資料。

本稿定義 **概念、訊號、風險與 MVP 落地順序**；實作評分與 schema 須待規則穩定後另開 Phase。

## 2. 設計原則（與 AGENTS.md 對齊）

| 原則 | 對激勵制度的意涵 |
|------|------------------|
| 不暗中保存個別答案 | 品質分不得依「發起者是否看過期中票數」或 per-user option 紀錄計算 |
| 低信任不污染正式結果 | 被檢舉／停權／取消的問卷不得產生或回饋品質獎勵 |
| 排行不操弄答案方向 | 品質分 **不得** 用於收票中的票數、百分比、趨勢或答案方向訊號 |
| 收票中隱藏結果 | 「參與度」訊號僅能在 **統計期結束後** 納入品質評估（見 §5） |

## 3. 每日發起問卷基礎額度（依使用者等級）

以下為 **建議草稿等級**，用於 **每日可建立的問卷數量上限**（含已發布；草稿是否計入由實作 Phase 定義）。數字可隨營運調整，但須在 UI 與說明中透明揭露。

| 等級 | 建議名稱 | 每日可發起問卷數（草稿） |
|------|----------|---------------------------|
| **Lv.0** | 新加入／預設 | **3** 題／日 |
| **Lv.1** | 穩定參與 | **5** 題／日 |
| **Lv.2** | 良好紀錄 | **8** 題／日 |
| **Lv.3** | 高信任發起者 | **12** 題／日 |
| **Lv.4** | 平台認可發起者（嚴格審核） | **20** 題／日 |

### 3.1 等級如何取得（方向性，非 MVP 實作）

- **基礎：** 帳號年齡、完成個人檔案必要欄位、無近期嚴重違規。
- **提升：** 累積 **品質點數**、長期低檢舉率、公開結果期滿且未因下架／治理而失效的問卷比例。
- **降低：** 檢舉成立、洗版、誘導、協同灌分、取消／下架濫用等。

等級 **主要調整「基礎每日額度」**；額外臨時額度由 **品質點數** 消耗換取（§7）。

### 3.2 與生命週期的關係

- **收票中** 不得因「即時參與人數」在 UI 上激勵發起者（避免與 Phase 39 期中結果隱藏衝突）。
- **取消** 的問卷不計入有效發起成功次數的獎勵（§9）。
- **鎖定期內** 不得因發起者想衝發題量而繞過編輯限制（發題與編輯是不同維度，但治理狀態須一致）。

## 4. 什麼是「優質提問」？

**優質提問** 指：在符合平台規範與分類前提下，能讓 **符合資格** 的參與者理解題意、做出有意義選擇，且 **事後** 的公開聚合結果對社群有參考價值的問卷。

### 4.1 正向特徵（草案）

| 維度 | 說明 |
|------|------|
| **清楚** | 標題一句話可懂；選項互斥、涵蓋主要可能性；避免雙重否定與陷阱式措辭 |
| **中立** | 不預設「應該選哪個」；不在題幹夾帶煽動、羞辱或未證實斷言 |
| **適當分類** | 分類與內容一致；高敏感類別在 MVP **未正式開放**（見 AGENTS.md） |
| **資格合理** | 年齡／地區限制與題目相關且可理解；不濫用資格排除特定群體 |
| **非重複／非洗版** | 非短時間內語意幾乎相同的重複發題 |
| **可參與** | 統計期內有足夠且可信的正式投票參與（**截止後** 才評估，見 §5） |
| **事後有用** | 結果公開後，追蹤／回訪／正向回饋比例顯示內容對讀者仍有參考性 |
| **低風險治理** | 檢舉率低、未進入停權／系統封存 |

### 4.2 不屬於優質提問（草案）

- 標題黨、誘導點擊、刻意製造對立卻無清晰選項結構；
- 選項明顯引導單一答案或侮辱特定族群；
- 與已存在熱門問卷 **語意重複** 且無新意；
- 為衝每日額度而發的 **無意義** 或 **測試性** 大量問卷；
- 以協同帳號灌 **讚** 或灌 **追蹤** 卻無真實參與；
- 收票結束前即透過站外管道引導「先投某選項」以操弄（違反平台公平敘事）。

## 5. 不可只依「按讚」判定品質

**按讚（likes）** 容易受社交圈、時機、情緒與刷讚影響，且與 **收票中不得展示票數趨勢** 的產品精神不一致若作為即時排行訊號。

因此：

- 按讚 **可作為輔助訊號之一**，權重應 **低於** 結構化品質指標與 **事後** 參與訊號；
- **禁止** 僅憑讚數解鎖高額每日發題或高風險分類；
- UI 上避免「讚越多＝題目越好」的單一視覺排名（尤其收票中）。

## 6. 可能的品質訊號（多訊號、分階段採用）

以下訊號皆須在 **不違反隱私與 Raw Option Linkage Ban** 前提下，以 **聚合、延遲、去識別** 方式計算。標註 **MVP** 者為可先文件化／UI 示意者；標註 **Later** 者待 schema 穩定後實作。

| 訊號 | 說明 | 建議階段 |
|------|------|----------|
| **足夠參與** | 統計期結束後，符合資格的正式投票數達門檻（門檻依分級顯示規則設計，非 raw counter 外洩） | Later |
| **正向回饋比例** | 結果公開後，「有幫助／清楚」類回饋 vs 負向回饋（需定義 UI 與防刷） | Later |
| **追蹤／結果關注** | 結果追蹤轉換率、結果頁回訪（**不得** 與個別 option 綁定） | Later |
| **低檢舉率** | 檢舉成立數／檢舉總量比；成立則重罰 | Later |
| **用語清楚** | 規則式：標題長度、選項數、禁用詞、重複 n-gram 偵測 | MVP（規則示意）／Later（自動化） |
| **非誤導** | 與分類、資格、統計期文案一致；無假截止語意 | MVP（人工＋規則） |
| **非重複／非 spam** | 發起者短期相似度、全站相似度 | Later |
| **結果事後有用性** | 公開後一段時間內的閱讀完成度、分享品質（非刷量） | Later |

**收票期間禁止作為品質加分的訊號：**

- 即時票數、百分比、排名、趨勢；
- 發起者儀表板上的期中參與人數；
- 任何可讓發起者或觀眾推測「目前領先選項」的資料。

## 7. 品質點數／信用點數（概念）

**品質點數**（亦稱 **信用點數**，名稱待 UI 定稿）代表：平台對你 **過往發題與問卷完整生命週期表現** 的累積評價，**不是** 可交易貨幣，**不可** 轉讓。

### 7.1 獲得（方向性）

- 問卷 **正常走完**：統計期結束 → 結果公開 → 鎖定期滿且無嚴重檢舉成立；
- 事後回饋與參與訊號達標（§6）；
- 長期低檢舉、無洗版紀錄。

### 7.2 扣除或凍結

- 檢舉成立、誤導、重複 spam、協同灌分；
- **取消** 問卷（尤其臨近截止前惡意取消）；
- **下架** 後若涉違規或爭議內容；
- 政治／高風險類別違規（§10）。

### 7.3 品質點數的用途（草案）

| 用途 | 說明 |
|------|------|
| **暫時提高發題額度** | 例如：消耗 N 點，今日額外 +1 題（設每日消耗上限） |
| **提升信任等級** | 累積達標可升 Lv.1→Lv.4，提高 **基礎** 每日額度 |
| **解鎖額外發起者工具** | 例如：進階截止設定、草稿範本、批次匯出 **聚合** 報表（仍須 display-safe） |
| **未來高風險分類資格** | 僅 **未來** 若開放高敏感類別時，作為 **必要門檻之一**（非 MVP） |

所有用途須在說明頁透明列出；**不得** 用點數購買「收票中看票數」或「指定選項曝光」。

## 8. 濫用風險與緩解

| 風險 | 描述 | 緩解方向 |
|------|------|----------|
| **讚農場（like farming）** | 多帳號互讚、無真實投票 | 讚權重低；異常圖偵測；讚不直接換額度 |
| **標題黨（clickbait）** | 聳動標題、內容與選項不符 | 檢舉＋規則偵測；事後回饋；扣點 |
| **協同灌分（coordinated boosting）** | 團伙集中追蹤／分享／假參與 | 速率限制；裝置／行為聚合（不存 option 連結）；人工複核 |
| **低品質病毒題** | 短期爆量但無參考價值 | 參與門檻延遲計分；鎖定期後才給大額獎勵 |
| **政治／爭議操弄** | 利用問卷製造輿論假象 | 高風險類別嚴格規則；MVP 不正式開放；治理停權 |

## 9. 安全規則（強制）

以下規則 **優先於** 任何品質加分邏輯：

| 情境 | 規則 |
|------|------|
| **被檢舉且審核中** | **暫停** 品質點數累積與額度消耗；UI 可顯示「審核中」 |
| **檢舉／治理成立** | **不得** 產生品質獎勵；已發點數可 **追回或凍結** |
| **已取消問卷** | **不得** 產生品質獎勵；不計入「成功發起」統計 |
| **已下架問卷** | 若下架原因為違規：**追回**；若為正常下架：僅 **事後已公開期間** 的表現可部分計入（由政策細化，避免鼓勵先上架再下架洗版） |
| **未公開結果** | 收票中、取消、未 reveal 者 **不得** 用參與度發獎 |
| **政治／高風險類別** | MVP **未正式開放**；未來須更高等級、更長鎖定、更嚴檢舉門檻與可能 Dual-Admin 治理 |

激勵系統 **不得** 鼓勵發起者透過 **取消** 逃避劣質問卷紀錄，亦 **不得** 在 **鎖定期** 內以任何方式變相修改已公開結果以換取點數。

---

## 10. 政治／高風險與分類政策

- MVP 依 `AGENTS.md`：**高敏感分類不正式開放**。
- 品質激勵 **不得** 成為繞過分類限制的捷徑（例如用 Lv.4 自動開放政治類）。
- 若未來開放，建議額外要求：更高 Lv、更長公開鎖定期、專用檢舉 SLA、可能 **不得** 給予「額外發題額度」獎勵，僅允許 **受限工具**。

## 11. UI／文案建議（支援 Public MVP 設計）

在 **未實作計分** 前，UI 可採 **「說明＋佔位」**：

| 位置 | 建議內容 |
|------|----------|
| 發起問卷頁 | 簡短「優質提問小提醒」：清楚、中立、選項互斥、勿重複發題 |
| 發起者儀表板 | 顯示 **等級 Lv.0** 與 **今日剩餘額度 3/3**（mock）；品質點數顯示「即將推出」 |
| 說明／FAQ | 連結 `docs/www-project-public-faq-draft-v1.md` 對應章節 |
| 收票中 | **不顯示**「再 X 票升級」或讚數排名 |

範例文案（非最終）：

> 發起清楚、中立的問卷，有助於提升你的發起者等級與每日額度。收票期間平台不顯示票數，請勿以聳動標題誤導參與者。

## 12. MVP 建議（實作順序）

| 階段 | 建議 | 不做 |
|------|------|------|
| **MVP（現在）** | 在本 repo **文件化** 本政策；Public MVP UI **展示** 等級／額度／品質點數 **概念**（mock 資料）；發起頁放「優質提問」提示 | 真實評分、DB 欄位、API、自動升降級 |
| **Phase A** | 固定 **每日硬上限**（僅 Lv.0=3），無點數 | 複雜多訊號模型 |
| **Phase B** | 引入 **品質點數** 與 **延遲訊號**（截止後）；檢舉聯動停獎 | 收票中即時榜單 |
| **Phase C** | 等級 Lv.1–4、額外工具、治理與高風險門檻 | 僅按讚排序 |

**Schema／migration 原則：** 僅在 §4–§9 規則穩定、與 Phase 39/40 生命週期欄位對齊後，再新增例如 `creator_level`、`daily_poll_quota`、`quality_points_balance`、`quality_event_log`（**不得** 含 user–option 連結）等表；且須通過隱私與 AGENTS.md 審查。

## 14. 開放問題（待產品定案）
