## 1. 目的

大家想知道（WWW Project）需要一套 **可理解、可稽核、可漸進落地** 的使用者信任等級，以區分：

- 訪客瀏覽與已註冊參與者的能力邊界；
- 一般問卷發起與 **政治／高風險** 等受控主題的門檻；
- 濫發、刷票、誤導與協同操弄的風險控制。

本稿為 **第一版政策草稿**，供產品、設計與工程對齊；**不等於** 已上線的帳號系統或 API 行為。

---

## 2. 設計原則

| 原則 | 對信任等級的意涵 |
|------|------------------|
| 收票中不洩漏結果 | **所有等級**（含發起者）在 collecting 期間 **不得** 查看票數、百分比、排名、趨勢或進度 |
| 誠實邊界 | UI 可預覽流程，但須標示「預覽模式／尚未開放／不儲存」 |
| 信任不等於免審 | 等級提升 **不能** 繞過政治／高風險類別的 **事前審核** |
| 品質不只按讚 | 「優質題目」**不是** 按讚數決定；以 **投票後品質回饋問卷**（多選標籤）為主訊號之一（見 §12） |
| 不實作即不承諾 | 手機驗證、裝置綁定、通知、真實額度計數等 **另 Phase 實作** |

---

## 3. 信任等級總覽

| 等級 | 名稱 | 摘要 |
|------|------|------|
| **Lv.0** | 訪客 | 瀏覽與查看已公開結果；不可投票、發起、關注、回饋 |
| **Lv.1** | 註冊用戶 | 完成基本註冊欄位後可投票、關注、發起一般問卷（低額度） |
| **Lv.2** | 可信註冊用戶 | 較高發起額度；可參與較高信任問卷；可發起較敏感但非高風險主題 |
| **Lv.3** | 優質用戶 | 限定期間資格；更高額度與能見度；參與品質回饋迴路 |
| **Lv.4** | 高信任（未來） | 長期穩定貢獻與更強驗證；最高額度草案；仍受高風險審核 |

---

## 4. Lv.0 — 訪客

**身分：** 未註冊、未登入。

**必要條件：** 無。

**可以做：**

- 瀏覽／搜尋問卷基本資訊（標題、說明、狀態、公開時間等 **非收集中票數** 欄位）；
- 查看 **已公開** 的彙總結果（依結果頁 display-safe 規則）。

**不可以做：**

- 投票；
- 建立問卷；
- 關注結果通知；
- 提供結果品質回饋；
- 查看 **收票中（collecting）** 的期中結果、票數、百分比、排名、趨勢或進度。

---

## 5. Lv.1 — 註冊用戶

### 5.1 註冊必填（草稿）

| 欄位 | 說明 |
|------|------|
| Email | 帳號識別與通知（未來） |
| 暱稱 | 對外顯示名稱 |
| 出生年／月 | **不含** 完整生日；用於年齡資格區間 |
| 居住縣市／地區 | 地區資格與內容在地化 |
| 性別 | 含「**不透露**」選項 |

### 5.2 可以做

- 在符合資格的問卷 **投票**；
- **關注** 問卷結果（站內通知，實作另 Phase）；
- 建立 **一般分類** 問卷，受 **低每日發起額度** 限制；
- 投票後可選填 **品質回饋問卷**（多選標籤，見 §12；實作另 Phase）。

### 5.3 不可以做

- 建立 **政治／高風險** 類別問卷；
- 繞過問卷設定的 **投票資格** 限制；
- 查看 collecting 期中結果或發起者期中儀表板數字。

---

## 6. Lv.2 — 可信註冊用戶

### 6.1 可能的信任訊號（草稿，非全部必填）

- 手機驗證（未來實作）；
- 裝置綁定（未來實作）；
- 帳號建立時間／帳齡；
- 近期行為正常（投票、發起、互動模式）；
- 檢舉／spam 風險低。

### 6.2 可以做

- **較高** 每日發起問卷額度（相對 Lv.1）；
- 參與需較高信任門檻的問卷；
- 建立 **較敏感但非政治／高風險** 主題（仍可能觸發審核）；
- 提供 **較強** 的品質回饋訊號權重（實作另 Phase）。

### 6.3 不可以做

- 自動免審發布政治／高風險問卷；
- 查看 collecting 期中結果。

---

## 7. Lv.3 — 優質用戶

### 7.1 取得條件（草稿）

- **必須** 已是 Lv.2；
- 近期有 **高品質貢獻**（見 §7.2）；
- 資格 **有期限**，草稿建議：**21 天**；到期需重新評估。

### 7.2 品質評估訊號（不可只靠按讚）

**核心機制** 為 §12 的 **投票後品質回饋問卷**（類似外送完成後對司機的簡短評價，但評的是 **題目本身**）。按讚／upvote **不得** 作為唯一或主要條件。

須綜合多項訊號，例如：

| 訊號類型 | 說明 |
|----------|------|
| **投票後品質回饋** | 投票者對題目的多選正向標籤比例；含一項柔和中性標籤（§12.2） |
| 參與率 | 統計期結束後的有效參與，**非** 收票中票數 |
| 檢舉率 | 低檢舉、低違規 |
| 題目清晰度 | 標題／選項是否可理解（可與回饋標籤「題目清楚」呼應） |
| 非誤導措辭 | 避免誘導、偽裝中立 |
| 非重複／spam | 與既有問卷去重 |
| 公開後仍有用 | 結果揭曉後對讀者仍有資訊價值 |
| 按讚／upvote | **僅可為輔助**，權重低於結構化品質回饋；**禁止** 僅憑讚數認定優質 |

### 7.3 可以做

- **更高** 每日發起額度；
- 優質一般問卷可獲 **較高能見度**（探索／推薦，實作另 Phase）；
- 參與 **品質回饋迴路**（投票後多選回饋、累積優質題目紀錄，實作另 Phase）。

### 7.4 不可以做

- 自由發布政治／高風險問卷；
- 繞過平台審核；
- 查看 collecting 期中結果。

---

## 8. Lv.4 — 高信任（未來）

**狀態：** 規劃中，**不在 MVP 實作範圍**。

**可能方向：**

- 長期維持 Lv.3 且違規紀錄低；
- 多題高品質問卷累積；
- 更強身分／裝置驗證；
- **最高** 每日發起額度草案。

**即使 Lv.4** 亦 **不可** 在未經審核下自由發布政治／高風險主題。

---

## 9. 政治／高風險類別（受控區）

此類別視為 **受控區（controlled area）**，規則優先於單純信任等級：

1. **較高信任門檻** 才可 **申請** 發起（非 Lv.1 預設能力）；
2. **事前審核（pre-review）** 通過後方可發布；
3. 須偵測 **委婉語、諧音、色碼誘餌、間接政治引導** 等規避手法；
4. **信任等級 alone 不能 bypass 審核** — 含 Lv.3、Lv.4。

## 11. 與投票／結果隱私的關係

- **任何等級** 的使用者與發起者，在 **collecting** 期間均不得取得期中票數、百分比、排名、趨勢或進度（見 Phase 39）。
- 信任等級 **不授予** 「提前看結果」特權。
- 不符合投票資格者（含 Lv.0）仍可 **關注** 結果公開通知（Lv.1+ 完整能力；訪客可引導註冊）。

## 12. 投票後品質回饋與「優質題目」判定（草案）

平台應 **鼓勵正向貢獻習慣**，不只懲罰濫用。使用者在 **完成投票後**，可選擇性回答一則簡短的 **品質回饋問卷**，協助評價 **這一題問卷寫得好不好**（不是評個別選項、也不是公開羞辱發起者）。

### 12.1 投票後品質回饋問卷

**時機：** 使用者送出投票後（可略過；不應造成高摩擦）。

**題幹（建議）：** 「你覺得這個問題如何？可複選」

**正向多選標籤（建議，可調整文案）：**

| 標籤 | 意涵（草案） |
|------|----------------|
| 題目清楚 | 讀得懂、選項好選 |
| 無刻意誤導 | 未明顯誘導或陷阱式措辭 |
| 有參考價值／值得省思 | 事後仍值得看結果或討論 |
| 新奇有趣 | 有新鮮感但不等於聳動 |
| 中立客觀 | 語氣相對平衡 |
| 我想看結果 | 願意追蹤／關注公開結果 |

**柔和中性訊號（僅一項，非公開羞辱、非正式檢舉）：**

- **「有點無言／不知道該怎麼說」** — 表達困惑或難以評價，**不是** 貼在問卷上的負面標籤，也 **不是** 對發起者的人身攻擊入口。

**產品原則：**

- 以 **正向多選** 為主，營造建設性、低對立氛圍；
- **避免** 明確負面標籤清單（如「爛題」「垃圾」「故意引戰」等），以免激化情緒與站內衝突；
- 正式違規仍走 **檢舉／治理**，與品質回饋分軌。

**隱私：** 回饋以 **聚合** 方式納入品質評估；不公開「誰選了哪個標籤」。

### 12.2 「優質題目」判定

一則問卷要被認定為 **「優質題目」**（用於能見度、發起者信任累積、Lv.3 相關訊號等），須同時滿足 **方向性** 條件（門檻數字另 Phase 定義）：

1. 收到 **足夠多** 投票者的品質回饋（避免小樣本操弄）；
2. **正向標籤** 的聚合比例達標，且通過安全／品質閾值（非誤導、非 spam、檢舉率低等）；
3. **按讚／upvote 不是唯一條件**，亦不可作為主要條件；
4. 品質回饋 **不是** 純人氣榜 — 須與參與率、事後有用性等訊號並看。

**柔和中性標籤的否決規則：**

- 若 **「有點無言／不知道該怎麼說」** 在回饋者中成為 **多數（majority）**，該問卷 **不得** 認定為「優質題目」（即使票數或按讚偏高）。
- 此標籤 **不得** 對外公開為「羞恥章」或「劣質認證」。

### 12.3 可信任發起者（習慣養成）

發起者可透過累積 **足夠多** 通過門檻的 **優質題目**，取得或維持 **較高信任狀態**（如 Lv.2→Lv.3 的訊號之一）。目的在於獎勵：

- 更清楚的題目與選項；
- 較少誤導性措辭；
- 對讀者更有用的公開結果；
- 較少洗版、rage-bait 或重複 spam。

**未實作聲明：** 回饋問卷、聚合計分、優質題目徽章與信任升降 **均不在 MVP 實作**；僅政策與 UI 預覽文案。

## 13. Public MVP 文案邊界（Phase 45）

產品預覽頁面應：

- 使用「**預覽模式**」「**目前為產品預覽，資料不會儲存**」「**此功能尚未開放**」「**登入後可使用**」等用語；

- **不** 暗示假功能已上線。
